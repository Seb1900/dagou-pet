# 大狗桌宠隐私说明

生效日期：2026-07-23

大狗桌宠由 Seb1900 开发并免费发布。本说明适用于 GitHub 仓库 `Seb1900/dagou-pet` 提供的官方版本。

## 全局按键监听

应用使用 `uiohook-napi` 接收全局键盘的按下和松开事件，以便用户在其他窗口输入时驱动桌宠声音和动作。

- 事件在本机处理。
- 为防止系统漏发松键事件造成声音卡住，应用只在按键被视为按住期间读取该物理键当前是否仍按下；状态不会保存或上传。
- 应用不会记录或上传输入字符、输入内容、窗口标题、进程名称、剪贴板内容或输入历史。
- Renderer 只会收到按压编号、按键角色、音阶步进和声像。
- “暂停监听”会停止全局键盘 Hook。

## 本地数据

应用在 Electron 的用户数据目录保存 `settings.json`，内容包括音量、桌宠大小、镜像、按键绑定和窗口位置等设置。更新设置时会保留一个本地备份。应用不建立用户账户，也不使用广告、分析或行为追踪服务。

卸载应用时，Windows 可能保留用户数据目录。用户可以自行删除该目录来清除设置。

## 网络请求

NSIS 安装版会在启动后延迟检查 GitHub Releases 是否存在新版，也可由用户在“关于”页手动检查。免安装版只在用户点击更新入口后打开官方发布页。

更新请求会向 GitHub 传递网络通信所需的信息，例如 IP 地址、应用版本、操作系统信息、请求时间和 User-Agent。其处理方式受 GitHub 隐私声明约束。应用不会把全局按键事件附加到更新请求中。

点击“项目地址”会用系统浏览器打开 GitHub；点击“反馈问题”会打开飞书表单。应用不会读取浏览器中的登录状态或页面内容。

## 联系方式与变更

普通隐私问题可在[官方仓库 Issues](https://github.com/Seb1900/dagou-pet/issues)反馈。涉及安全漏洞时请按 [SECURITY.md](SECURITY.md) 使用私密渠道。

本说明发生实质变化时，会在 `CHANGELOG.md` 和对应 GitHub Release 中记录。
